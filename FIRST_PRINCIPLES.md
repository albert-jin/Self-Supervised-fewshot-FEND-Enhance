# First Principles

## P1. Minimal Verification, Maximum Implementability

### Permanent Prompt
在当前设备条件下，不执行测试与大规模运行验证。请以“实现闭环”为核心目标完成代码：  
1. 每个子模块（或子智能体）只负责自己清晰边界内的实现，并确保接口可被主流程直接调用；  
2. 代码以可读、可维护、与实验逻辑一致为最高优先级，保证在合理环境下具备基本可运行性；  
3. 注释仅保留与实验设计、数据流程、损失函数或复现实验相关的信息；  
4. 避免任何“AI 助手口吻”或“自动生成痕迹”文本，不写与实验无关的说明性冗余注释；  
5. 交付时明确假设、输入输出与模块职责，确保后续可继续迭代复现。  

### Execution Rule
该原则默认适用于本项目后续所有开发任务，除非用户显式覆盖。
